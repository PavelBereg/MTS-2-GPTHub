# Быстрый старт

Развертывание проекта MTS AI Assistant локально максимально упрощено благодаря Docker Compose. Это рекомендуемый способ запуска, так как он автоматически поднимает сам FastAPI-бэкенд, базу данных Qdrant и (опционально) клиент OpenWebUI.

## Запуск через Docker

1. **Клонируйте репозиторий проекта:**
   ```bash
   git clone <repo_url>
   cd MTS-hack
   ```

2. **Настройте переменные окружения:**
   Скопируйте пример файла окружения и добавьте ваш MWS API ключ.
   ```bash
   cp .env.example .env
   ```
   Откройте `.env` и впишите ключ в переменную: `MWS_API_KEY=ваш_секретный_ключ`

3. **Запустите контейнеры:**
   ```bash
   docker-compose up -d --build
   ```

### Что запустится:
- **FastAPI бэкенд**: [http://localhost:8000](http://localhost:8000) (Swagger UI: [http://localhost:8000/docs](http://localhost:8000/docs))
- **Qdrant (Vector DB)**: Сохраняется локально в директории `qdrant_data/`. Контекст пользователя сохраняется между перезапусками.
- **OpenWebUI**: Клиентский интерфейс, доступен на порту `3000` (если включен в docker-compose.yml).

## Интеграция с OpenWebUI

Если вы запускаете свой собственный экземпляр OpenWebUI, вы можете легко подключить к нему наш бэкенд:

1. Перейдите в настройки OpenWebUI: **Settings > Connections > OpenAI API**.
2. Добавьте новое подключение со следующими параметрами:
    - **Base URL**: `http://localhost:8000/v1`
    - **API Key**: Любое значение (мы используем ключ сервера, поэтому клиентский ключ игнорируется).
3. Сохраните настройки. В интерфейсе чата у вас появятся новые доступные модели, например: `AUTO (Умный роутер МТС)`.

> [!TIP]
> При запуске через наш `docker-compose.yml` OpenWebUI настраивается автоматически через переменные окружения, отключая встроенный RAG в пользу нашей долгосрочной памяти.

## Запуск без Docker (для разработки)

Если вам нужно разрабатывать бэкенд вне контейнера:

1. Поднимите Qdrant в контейнере: `docker run -p 6333:6333 qdrant/qdrant`
2. Создайте [виртуальное окружение](https://docs.python.org/3/library/venv.html) и активируйте его.
3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
4. Убедитесь, что `.env` файл заполнен корректно и содержит:
   ```env
   QDRANT_URL=http://localhost:6333
   ```
5. Запустите приложение:
   ```bash
   uvicorn app.main:app --reload --port 8000
   ```

## Просмотр документации (MkDocs)

Проект содержит расширенную документацию в формате Markdown. Вы можете сгенерировать красивый статичный сайт для удобного чтения:

1. Установите MkDocs и тему Material:
   ```bash
   pip install mkdocs-material
   ```
2. Запустите сервер документации:
   ```bash
   mkdocs serve -a localhost:9000
   ```
3. Откройте в браузере: [http://localhost:9000](http://localhost:9000)

> [!IMPORTANT]
> Мы используем порт **9000** для документации, чтобы он не конфликтовал с основным бэкендом (порт 8000) и БД (порт 6333).

 ## Устранение неполадок (Troubleshooting)
 
 При возникновении ошибок выполните следующие действия:
 
 1. **Проверка логов:**
    Просмотрите вывод бэкенда для поиска конкретных ошибок API или исключений Python:
    ```bash
    docker-compose logs -f backend
    ```
 
 2. **Проверка доступности портов:**
    Убедитесь, что порты не заняты другими приложениями:
    - `netstat -ano | findstr :8000` (Windows)
    - `ss -tulpn | grep :8000` (Linux)
 
 3. **Связь между контейнерами:**
    Если Backend не может подключиться к Qdrant, проверьте правильность `QDRANT_URL` в `.env`. Он должен соответствовать имени сервиса в `docker-compose.yml`.
 
 4. **Ошибки MWS API:**
    Код ответа `401` означает невалидный API ключ. Код `404` может указывать на отсутствие выбранной модели в вашем профиле MWS.
