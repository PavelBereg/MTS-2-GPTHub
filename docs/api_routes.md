# Роутинг и API

Эндпоинты нашего приложения полностью совместимы с протоколом OpenAI (в частности `/v1/chat/completions`), что позволяет использовать его как Drop-in replacement для любых UI клиентов. Маршрутизация запросов происходит посредством выбора "виртуальной модели".

## `POST /v1/chat/completions`

Главный эндпоинт, обрабатывающий большинство запросов. 

### Виртуальные режимы ("Модели")

Вместо того чтобы отправлять запросы напрямую в LLM, сервер принимает кастомные значения в поле `model` и маршрутизирует логику:

| Режим (model) | Описание пайплайна |
|:---|:---|
| `AUTO (Умный роутер МТС)` | Динамическая классификация. Анализирует запрос, определяет намерение пользователя и отправляет на наиболее подходящий пайплайн обработки (будь то генерация картинки, глубокий рисерч или простой чат). |
| `Обычный чат` | Классическое потоковое (SSE) общение с мощной моделью MWS без применения инструментов. Идеально для написания кода, текстов или простого общения. |
| `Поиск + Чат` | В этом режиме агент наделен инструментами `web_search_tool` and `scrape_website_tool` для поиска актуальной информации и ответов на основе свежих данных. |
| `Deep Research` | Многошаговый процесс исследования. Агент самостоятельно декомпозирует запрос на подвопросы, ищет информацию по каждому и компилирует единый объемный отчет. Пользователь может управлять глубиной через `[глубина: N]`. |
| `📊 Аналитика и Графики` |intent: `data`. Подключает инструмент `execute_python` (Python Sandbox) для визуализации данных через pandas/matplotlib. |
| `🌐 Генератор Сайтов` | intent: `website`. Активирует пайплайн создания прототипов веб-интерфейсов и лендингов. |
| `📄 Документы Word` | Поддерживает генерацию структурированных `.docx` файлов на основе описания. |

> [!NOTE]
> Системные и служебные запросы OpenWebUI (генерация заголовков, тегов) автоматически перехватываются сервером (через `is_utility_request`) и направляются на самую быструю и дешевую LLM `llama-3.1-8b-instruct`.

## Обработка Изображений (VLM)

Если параметр `messages` содержит вложения-изображения (VLM — Vision Language Model), запрос перехватывается, минует базовую текстовую обработку и перенаправляется на выделенную нейросеть технического зрения `qwen2.5-vl-72b` с промптом для глубокого анализа визуального содержимого.

## Аудио (ASR) Эндпоинты

Расположены в `app/api/audio_routes.py`.

* **`POST /v1/audio/transcriptions`**
  Совместимый порт для получения аудио (микрофон OpenWebUI). Отправляет данные на сервис распознавания `whisper-turbo-local` и возвращает распознанный текст.
  Лимит файла составляет 25 MB.

* **`POST /v1/audio/chat`**
  Законченный pipeline для интеграции из сторонних фронтендов, которые не поддерживают нативное ASR. Принимает аудио, распознает его, передает в LLM и возвращает ответ.

## PPTX Помощник

* **`POST /v1/pptx/generate`**
  Самостоятельный endpoint для построений презентаций. 
  Принимает JSON с параметрами (тема, количество слайдов), генерирует структуру через LLM, отрисовывает слайды в файлы `.pptx` (используя пакет поддержки python-pptx) и возвращает мета-информацию и ссылку для скачивания файла (см. `pptx_routes.py`).
